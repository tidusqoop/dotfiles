# htop 사용법

> `top`의 상위 호환. CPU/메모리 사용량, 프로세스 목록을 컬러풀한 인터랙티브 UI로 보여주는 모니터링 도구.

## 설치

```bash
brew install htop
```

> 모든 프로세스를 보려면 `sudo htop`으로 실행해야 한다.

## 기본 사용법

```bash
# 실행
htop

# 모든 프로세스 표시 (root 권한)
sudo htop

# 특정 사용자 프로세스만
htop -u mshyeon

# 특정 PID만
htop -p 1234

# 트리뷰로 시작
htop -t
```

## 화면 구성

```
┌─────────────────────────────────────────────┐
│ CPU [||||||||     25%]  Mem [||||||    1.2G] │  ← CPU/메모리 미터
│ CPU [||           8%]   Swp [          0K]  │
├─────────────────────────────────────────────┤
│ PID  USER  PRI  NI  VIRT  RES  SHR  S  ... │  ← 프로세스 목록
│ 1234 root   20   0  100M  50M  10M  S  ... │
│ 5678 user   20   0  200M  80M  20M  R  ... │
├─────────────────────────────────────────────┤
│ F1Help F2Setup F3Search F4Filter F5Tree ... │  ← 기능 키
└─────────────────────────────────────────────┘
```

## 키 조작

### 탐색

| 키 | 동작 |
|----|------|
| `↑` / `↓` | 프로세스 선택 |
| `Space` | 프로세스 태그 (다중 선택) |
| `/` | 프로세스 이름 검색 |
| `\` | 프로세스 필터링 (검색과 달리 매칭된 것만 표시) |
| `t` | 트리뷰 토글 (부모-자식 관계 표시) |
| `H` | 사용자 스레드 숨기기/표시 |
| `K` | 커널 스레드 숨기기/표시 |

### 정렬

| 키 | 정렬 기준 |
|----|----------|
| `P` | CPU 사용률 |
| `M` | 메모리 사용률 |
| `T` | 실행 시간 |
| `>` / `<` | 다음/이전 컬럼으로 정렬 |
| `I` | 정렬 반전 |

### 프로세스 관리

| 키 | 동작 |
|----|------|
| `k` | 선택한 프로세스에 시그널 보내기 (kill) |
| `F9` | kill 메뉴 (시그널 선택) |
| `e` | 프로세스의 환경변수 보기 |
| `l` | 프로세스가 열고 있는 파일 보기 (lsof) |
| `s` | 프로세스의 시스템 콜 추적 (strace/dtrace) |

### 표시 설정

| 키 | 동작 |
|----|------|
| `F2` | 설정 화면 (미터, 컬럼 커스터마이징) |
| `u` | 특정 사용자의 프로세스만 표시 |
| `p` | 전체 경로 표시 토글 |
| `F5` | 트리뷰 토글 |
| `q` | 종료 |

## top과 비교

| 기능 | `top` | `htop` |
|------|-------|--------|
| 색상 | 없음 | 컬러풀 |
| 마우스 | 미지원 | 클릭으로 정렬/선택 |
| 스크롤 | 제한적 | 자유로운 상하좌우 스크롤 |
| 트리뷰 | 없음 | `t` 키로 부모-자식 관계 표시 |
| 프로세스 검색 | 없음 | `/` 키로 즉시 검색 |
| 프로세스 필터 | 없음 | `\` 키로 필터링 |
| kill | PID 입력 필요 | 선택 후 `k` 키 |
| 설정 커스터마이징 | 제한적 | F2로 미터/컬럼 자유 편집 |

## 실전 활용 예시

### CPU를 많이 쓰는 프로세스 찾기

```bash
htop
# P 키를 눌러 CPU 사용률로 정렬
```

### 메모리 누수 의심 프로세스 확인

```bash
htop
# M 키를 눌러 메모리 사용률로 정렬
# RES 컬럼이 계속 증가하는 프로세스 확인
```

### 특정 프로세스 찾아서 종료

```bash
htop
# / 키로 프로세스 이름 검색 (예: "node")
# k 키로 시그널 전송 (15: SIGTERM, 9: SIGKILL)
```

### Docker 컨테이너 프로세스 확인

```bash
htop -t
# 트리뷰에서 containerd 하위 프로세스 확인
```

### 특정 서비스 모니터링

```bash
htop
# \ 키로 필터링 (예: "python")
# 해당 프로세스만 화면에 표시됨
```
