# Neovim

터미널 네이티브 코드 편집기. VS Code 에디터 부분을 대체.

## VS Code와의 비교

| | VS Code | Neovim |
|---|---|---|
| 동작 방식 | 마우스 + 키보드 | 키보드 전용 (모달 편집) |
| 철학 | 기능을 메뉴에서 찾음 | 명령을 조합해서 실행 |
| 확장성 | Extension 마켓 | Lua 플러그인 생태계 |
| 리소스 | Electron 기반 (무거움) | 터미널 네이티브 (가벼움) |
| 원격 작업 | Remote SSH 확장 필요 | tmux 안에서 바로 사용 |

## 모달 편집

Neovim의 핵심. 모드에 따라 같은 키가 다른 동작을 함.

```
Normal 모드:  이동, 삭제, 복사 (기본 상태)
Insert 모드:  타이핑 (i 키로 진입, Esc로 복귀)
Visual 모드:  텍스트 선택 (v 키로 진입)
Command 모드: 명령 실행 (: 키로 진입)
```

### 왜 모달인가

일반 에디터: 모든 키가 항상 타이핑 → 편집 명령에 Ctrl/Alt 조합 필요
Neovim: Normal 모드에서 모든 키가 명령 → 조합 없이 단일 키로 강력한 편집

```
예: "단어 삭제"
VS Code:  Ctrl+Shift+K 또는 더블클릭 → Delete
Neovim:   dw (delete word)

예: "이 함수명 전체 치환"
VS Code:  Ctrl+H → 입력 → 하나씩 또는 전체 치환
Neovim:   :%s/oldName/newName/g
```

## AI 시대에 Neovim이 맞는 이유

- 편집 자체를 AI에게 맡기는 비율이 높아짐
- 에디터 역할이 "코드 확인/미세 수정"으로 축소
- 가볍게 열어서 확인하고 닫는 패턴에 적합
- tmux pane 안에서 에디터와 agent가 나란히 동작

## 시작 전략

### 프리셋 사용 권장

처음부터 설정하면 시간이 많이 걸림. 프리셋으로 시작.

| 프리셋 | 특징 |
|---|---|
| **LazyVim** | 가장 인기. 합리적 기본값, 쉬운 커스터마이징 |
| AstroNvim | 풍부한 UI, VS Code에 가까운 경험 |
| NvChad | 빠르고 가벼움, 미니멀 |
| kickstart.nvim | 공식 최소 템플릿, 학습용 |

### 점진적 전환

```
1주차: vimtutor 완료 (내장 튜토리얼, 30분)
       → 설정 파일(.yaml, .toml) 편집에만 사용

2주차: 기본 이동과 편집 익숙해지기
       → hjkl 이동, w/b 단어 이동, dd 삭제, yy 복사

3주차: 검색과 치환
       → / 검색, :%s 치환, * 단어 검색

4주차~: 플러그인 활용
       → telescope(fuzzy finder), treesitter(구문 강조), LSP(자동완성)
```

## 핵심 단축키 (최소한)

### 이동

```
h j k l     # 좌 하 상 우
w / b       # 다음/이전 단어
0 / $       # 줄 처음/끝
gg / G      # 파일 처음/끝
Ctrl-d/u    # 반 페이지 아래/위
```

### 편집

```
i / a       # 커서 앞/뒤에서 insert 모드
o / O       # 아래/위에 새 줄 추가 후 insert
dd          # 줄 삭제
yy          # 줄 복사
p           # 붙여넣기
u           # undo
Ctrl-r      # redo
```

### 파일

```
:w          # 저장
:q          # 종료
:wq         # 저장 후 종료
:e!         # 파일 다시 로드 (외부 변경 반영)
:e filename # 다른 파일 열기
```

## 현실적 고려

- **학습 곡선**: 1-2주는 생산성 하락 각오
- **근육 기억**: 익숙해지면 마우스보다 빠름, 그 전까지는 느림
- **플러그인 관리**: 가끔 업데이트로 깨지는 경우 있음
- **VS Code 병행**: 전환 초기에는 복잡한 작업은 VS Code, 간단한 편집은 neovim
