# CLI 중심 개발환경 가이드

VS Code에서 CLI/Shell 중심 환경으로의 전환 전략. DevOps + AI agent 시대에 맞춘 구성.

## 아키텍처 개요

```
┌─ Ghostty ─────────────────────────────────────────┐
│  터미널 에뮬레이터 (화면을 그리는 역할)              │
│                                                    │
│  ┌─ tmux ───────────────────────────────────────┐  │
│  │  세션/창 관리자 (작업 공간을 나누는 역할)       │  │
│  │                                               │  │
│  │  ┌─ neovim ─┐  ┌─ lazygit ─┐  ┌─ claude ─┐  │  │
│  │  │ 코드 편집  │  │ Git 관리   │  │ AI agent │  │  │
│  │  └──────────┘  └──────────┘  └──────────┘  │  │
│  └───────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────┘
```

각 레이어가 독립적인 책임을 가짐. VS Code는 이 모든 것이 하나로 합쳐진 형태.

## 레이어별 역할

### Ghostty — "화면"

글자를 빠르게 그리는 것. 그 이상도 이하도 아님.

- GPU 가속 렌더링
- 폰트, 색상, 투명도 등 시각적 요소 담당
- **비유**: 모니터. 무엇을 보여줄지는 관여하지 않고, 보여주는 품질만 담당

→ 상세: [ghostty.md](ghostty.md)

### tmux — "책상"

작업 공간을 조직하고, 작업을 영속적으로 유지.

- 세션 영속성 (터미널 닫아도, SSH 끊겨도 유지)
- 스크립팅 가능한 워크스페이스 구성
- 다중 모니터링/작업 동시 배치

→ 상세: [tmux.md](tmux.md)

### Neovim — "편집기"

터미널 안에서 동작하는 코드 편집기. VS Code 에디터 부분 대체.

- 키보드 전용 모달 편집
- Lua 플러그인 생태계
- 터미널 네이티브로 가벼움
- tmux pane 안에서 agent와 나란히 동작

→ 상세: [neovim.md](neovim.md)

### lazygit — "Git 조종석"

git 명령어를 TUI로 시각화. VS Code Source Control 패널 대체.

- 시각적 + 키보드 + 터미널 네이티브
- Interactive rebase, conflict 해결, stash 관리
- 학습 곡선이 완만 (1-2시간이면 기본 사용)

→ 상세: [lazygit.md](../cli-tools/lazygit.md)

## VS Code 기능 대체 매핑

| VS Code 기능 | CLI 대체 |
|---|---|
| 멀티 터미널 탭 | tmux panes/windows |
| 내장 터미널 | Ghostty |
| 파일 탐색기 | fzf + fd |
| 검색 | ripgrep (rg) |
| Git GUI | lazygit |
| 에디터 | neovim |
| 디버거 | 언어별 CLI 디버거 |

## AI Agent 시대의 이점

```
┌─────────────────────┬──────────────────────┐
│ claude (agent 1)    │ claude (agent 2)     │
│ - 기능 구현 중       │ - 테스트 작성 중      │
├─────────────────────┼──────────────────────┤
│ 로그/빌드 감시       │ git log / 상태 확인   │
└─────────────────────┴──────────────────────┘
```

- **다중 agent 동시 실행**: tmux pane마다 독립 Claude 세션
- **백그라운드 지속**: 터미널 닫아도 agent 계속 동작
- **감시 용이**: 자율 모드 agent 여러 개를 한 화면에서 모니터링
- **편집 역할 축소**: AI가 코드 작성 → 에디터는 확인/미세 수정용 → 가벼운 neovim이 적합

## 실제 작업 흐름

```bash
# 1. 프로젝트 세션 진입
tmux attach -s my-app

# 2. 멀티 pane 작업
#   [pane 1] nvim src/auth.py       → 코드 확인/편집
#   [pane 2] claude                 → AI에게 구현 요청
#   [pane 3] lazygit                → 변경사항 실시간 확인

# 3. Claude가 코드 작성 완료
#   → neovim에서 :e! 로 리로드하여 확인
#   → lazygit에서 diff 확인 → stage → commit → push

# 4. 세션 분리 (터미널 꺼도 유지)
# Ctrl-b d
```

## 전환 전략 (권장 순서)

```
1단계: tmux + lazygit  (난이도 낮음, 즉시 생산성)
       → 세션 관리와 Git 시각화부터 익숙해지기
       → VS Code 병행 가능

2단계: Ghostty 설정 강화  (테마, 키바인딩 정리)
       → tmux와의 키 충돌 해결

3단계: Neovim 도입  (난이도 높음, 장기 투자)
       → LazyVim 프리셋으로 시작
       → 처음엔 설정 파일 편집에만 사용
       → 점진적으로 주 에디터로 전환
```
